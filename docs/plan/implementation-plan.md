# Implementation Plan: Resort Management System

## Overview

Build a full-stack resort management system with room booking, boat tour booking, package deals, and payment processing for 5 user roles. Frontend: Next.js on Vercel. Backend: Express API deployed as Vercel serverless functions. Database: PostgreSQL + Prisma.

## Architecture Decisions

- **Express over NestJS** — simpler, less boilerplate, fits solo dev
- **Backend on Vercel serverless** — Express adapted via `@vercel/node`, same platform as frontend
- **Prisma ORM** — type-safe database access, auto-generated types shared between frontend and backend
- **JWT auth with refresh tokens** — stateless, works well with serverless
- **Omise/2C2P gateway** — industry standard in Thailand

## Dependency Graph

```
Prisma schema & migrations
    │
    ├── Shared types (generated by Prisma)
    │       │
    │       ├── Auth (JWT, OTP, Google OAuth)
    │       ├── User management (admin CRUD)
    │       ├── Room system (types, rooms, bookings, payments)
    │       ├── Boat system (types, boats, schedules, bookings, payments)
    │       ├── Package system
    │       ├── Reports
    │       └── Settings / contact info
    │
    └── Seed data
```

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Payment gateway integration complexity | High | Start with bank transfer + slip upload first (lower risk), add Omise/2C2P gateway in task 10 |
| Express on Vercel serverless (cold starts, request size limits) | Medium | Use serverless-http package; test early with a simple endpoint in Task 1 |
| Solo dev = slow progress | Medium | Vertical slicing ensures each task is independently deployable and demo-able |
| Google Maps API costs | Low | Set budget alerts; use static maps where possible |
| 5 user roles → complex UI state | Low | Shared dashboard layout with role-based component visibility; don't build 5 separate dashboards |
| LINE API approval in Thailand | Low | LINE Messaging API requires business verification; start with email-only, add LINE later |
